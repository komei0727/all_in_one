# ユビキタス言語集（全体）

## 概要

ユビキタス言語は、ドメインエキスパートと開発チームが共通で使用する言語です。
このドキュメントは、システム全体で使用される共通の用語を定義します。
各コンテキスト固有の用語は、それぞれのコンテキストのユビキタス言語集を参照してください。

## 基本用語

### ビジネス用語

| 用語       | 英語                    | 定義                                             | 使用例                       |
| ---------- | ----------------------- | ------------------------------------------------ | ---------------------------- |
| 食材       | Ingredient              | 料理に使用する材料。在庫管理の対象となる最小単位 | 「牛乳」「卵」「キャベツ」   |
| 在庫       | Stock/Inventory         | 現在保有している食材の数量                       | 「牛乳の在庫は2本」          |
| 賞味期限   | Best Before Date        | 品質が保証される期限。過ぎても安全性に問題はない | 「2025-01-25」               |
| 消費期限   | Use By Date/Expiry Date | 安全に消費できる期限。過ぎたら廃棄推奨           | 「2025-01-23」               |
| カテゴリー | Category                | 食材を分類するためのグループ                     | 「野菜」「肉・魚」「乳製品」 |
| 単位       | Unit                    | 食材の数量を表す単位                             | 「個」「g」「ml」「本」      |
| 保存場所   | Storage Location        | 食材を保管している場所                           | 「冷蔵」「冷凍」「常温」     |

### システム用語

| 用語     | 英語            | 定義                               | 備考                 |
| -------- | --------------- | ---------------------------------- | -------------------- |
| 登録     | Register/Create | 新しい食材をシステムに追加すること |                      |
| 更新     | Update          | 既存の食材情報を変更すること       |                      |
| 削除     | Delete          | 食材をシステムから除去すること     | 論理削除の場合もある |
| 検索     | Search          | 条件に合う食材を探すこと           |                      |
| 一覧表示 | List            | 複数の食材を表示すること           |                      |

## ドメイン概念

### エンティティ（Entity）

| 概念                   | 説明                           | 識別子       |
| ---------------------- | ------------------------------ | ------------ |
| 食材（Ingredient）     | 在庫管理の対象となる個々の食材 | 食材ID       |
| カテゴリー（Category） | 食材の分類                     | カテゴリーID |
| 単位（Unit）           | 数量の単位                     | 単位ID       |

### 値オブジェクト（Value Object）

| 概念             | 説明             | 構成要素        |
| ---------------- | ---------------- | --------------- |
| 数量（Quantity） | 食材の量を表す   | 数値 + 単位     |
| 金額（Money）    | 価格を表す       | 金額（円）      |
| 期間（Period）   | 日付の範囲を表す | 開始日 + 終了日 |

### ドメインイベント

| イベント             | 発生タイミング             | 含まれる情報                   |
| -------------------- | -------------------------- | ------------------------------ |
| 食材登録イベント     | 新しい食材が登録されたとき | 食材ID、食材名、登録日時       |
| 在庫更新イベント     | 食材の数量が変更されたとき | 食材ID、変更前数量、変更後数量 |
| 期限切れ間近イベント | 賞味期限が近づいたとき     | 食材ID、食材名、残り日数       |
| 在庫不足イベント     | 在庫が閾値を下回ったとき   | 食材ID、食材名、現在数量       |

## ビジネスルール用語

| 用語         | 定義                         | 例                                  |
| ------------ | ---------------------------- | ----------------------------------- |
| 期限切れ間近 | 賞味期限まで7日以内の状態    | 「この牛乳は期限切れ間近です」      |
| 在庫切れ     | 数量が0の状態                | 「卵が在庫切れです」                |
| 在庫不足     | 数量が設定した閾値以下の状態 | 「醤油が在庫不足です（残り100ml）」 |
| 適正在庫     | 過不足のない理想的な在庫量   | 「牛乳の適正在庫は2-3本」           |

## アクション用語

| アクション | 説明                       | 条件          |
| ---------- | -------------------------- | ------------- |
| 使い切る   | 食材の在庫を0にする        | 数量 > 0      |
| 買い足す   | 食材の在庫を増やす         | -             |
| 廃棄する   | 期限切れ等で食材を処分する | -             |
| 消費する   | 食材を使用して数量を減らす | 数量 > 消費量 |

## 状態用語

| 状態     | 説明                     | 表示 |
| -------- | ------------------------ | ---- |
| 新鮮     | 登録されたばかりの食材   | 🟢   |
| 通常     | 問題のない状態           | 🔵   |
| 要注意   | 期限が近い、在庫が少ない | 🟡   |
| 期限切れ | 賞味期限を過ぎた状態     | 🔴   |

## 単位の定義

### 個数系

- **個** - 個別にカウントできるもの（卵、りんご）
- **本** - 細長いもの（ネギ、ごぼう）
- **枚** - 平たいもの（肉、海苔）
- **袋** - 袋単位のもの（もやし、カット野菜）
- **パック** - パック単位のもの（納豆、豆腐）

### 重量系

- **g（グラム）** - 基本的な重量単位
- **kg（キログラム）** - 1000g
- **mg（ミリグラム）** - 0.001g

### 容量系

- **ml（ミリリットル）** - 基本的な容量単位
- **L（リットル）** - 1000ml

## 用語の使い分け

### 「賞味期限」vs「消費期限」

- **賞味期限**：品質が保たれる期限（調味料、缶詰など）
- **消費期限**：安全に食べられる期限（生鮮食品など）

### 「在庫」vs「数量」

- **在庫**：ビジネス文脈での表現
- **数量**：システム内部での表現

### 「削除」vs「廃棄」

- **削除**：システムから情報を消すこと
- **廃棄**：実際に食材を捨てること

## 注意事項

1. この用語集は随時更新される
2. 新しい機能追加時は必ず用語を定義する
3. ドメインエキスパートと合意した用語のみ追加する
4. コンテキスト固有の用語は各コンテキストで定義する
